> 版本：v1.2-design-spec
> 基于：v0.2 代码库分析
> 设计者：Design Lead

## 设计原则

### 1. 织布隐喻 — Weaving, Not Filing

织梦机的核心隐喻不是「文件夹-文件」，而是**织布机**。用户是在编织一个世界，不是在管理一个文件夹。

| 传统工具思维 | 织梦机思维 |
|-------------|-----------|
| 文件按目录分类 | 对象按正典等级编织 |
| 打开=编辑 | 打开=进入一个世界视图 |
| 保存=写入磁盘 | 修改=更新正典状态 |
| 搜索=文本匹配 | 导航=沿着引用网络行走 |

这个隐喻贯穿所有设计决策。

### 2. 渐进复杂度 — Surface Simplicity, Deep Power

新手看到的是书架+编辑器。高阶用户看到的是正典体系+画板推演+判断链。

- 默认视图对新手友好（一个作品=一个文档+一个画板）
- 高级功能（正典、判断、推演图）通过情境触发自然浮现，不需要先学概念
- UI 不做「设置面板」，而是「情境工具栏」

### 3. 正典优先 — Canon as First-Class Citizen

正典等级不是标签，而是**编织密度**的度量。每个操作要么提升正典，要么降低正典，要么记录判断。

- 正典状态是 UI 中**最显著的视觉元素**，不是藏在 dropdown 里的
- 新建对象自动分配「未收录」，需要用户主动收录才能成为设定
- 判断记录是操作的第一后果，不是事后日志

### 4. 差异化原则 — Not Another Note App

不是 Obsidian、Notion、Scrivener 的翻版。织梦机在以下维度差异化：

| 维度 | 织梦机 | Obsidian | Notion | Scrivener |
|------|--------|----------|--------|-----------|
| 核心单元 | 世界对象（8 种类型） | 笔记 | 页面 | 文档/文件夹 |
| 关联方式 | 正典等级+判断链+画板连线 | 双向链接 | 数据库关联 | 文件夹+标签 |
| 叙事工具 | 画板推演+时间线+正典仲裁 | 图谱 | 时间线视图 | 大纲+卡片 |
| 状态管理 | 6 种对象状态+判断操作 | 无 | 属性 | 进度标签 |
| 设计焦点 | 编织世界 | 知识管理 | 协作 | 长篇写作 |

## 1. 书架重设计

### 1.1 当前问题

- 无搜索栏：作品多时只能手动滚动查找
- 无筛选维度：不能按体裁（科幻/奇幻/武侠）或状态筛选
- 无排序：永远按创建顺序显示
- 无编辑：不能修改作品名、体裁、封面颜色
- 无删除：没有删除作品的功能
- 新建太粗糙：点击按钮直接创建「新作品」，无法选体裁和名称

### 1.2 设计方案

**整体布局重新组织：**

```
┌─────────────────────────────────────────────┐
│  作品书架            [+ 新建作品]  [排序▼]   │
│  [搜索...]   [全部体裁▼]  [全部状态▼]       │
├─────────────────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐      │
│  │觉醒纪元│ │星空彼岸│ │江湖行 │ │       │      │
│  │科幻   │ │奇幻   │ │武侠   │ │       │      │
│  │草稿中  │ │构思中  │ │修改中  │ │       │      │
│  │1.2万  │ │3800   │ │2.8万  │ │       │      │
│  │[⚙]    │ │[⚙]    │ │[⚙]    │ │       │      │
│  └──────┘ └──────┘ └──────┘ └──────┘      │
└─────────────────────────────────────────────┘
```

**设计亮点——「活封面」：**

每个作品卡片不再只是渐变背景，而是**活封面**——显示作品中正典等级最高的 3 个对象名称和类型图标，浮动在渐变背景之上。这给每本书一个可识别的「面孔」。

**交互细节：**

1. **搜索栏**：置于书架顶部，实时搜索作品名和体裁。输入时卡片网格即时过滤（无提交按钮，0.15s debounce）。

2. **筛选器**：两个 dropdown 筛选器——「全部体裁」和「全部状态」。体裁列表来自所有作品去重。状态列表固定（草稿中/构思中/修改中/已完成）。

3. **排序**：右上角排序 dropdown。选项：
   - 最近更新（默认）
   - 创建时间
   - 作品名 A-Z
   - 字数（多→少）

4. **作品操作菜单**：每张卡片右上角 `···` 按钮，hover 显示（点击展开）：
   - 编辑作品名
   - 更改体裁
   - 换封面色
   - 删除作品（确认对话框）
   - 导出

5. **编辑模式**：点击「编辑作品名」时，标题变为可编辑 input，按 Enter 确认、Esc 取消。体裁和封面在操作菜单中打开子弹窗选择。

6. **删除确认**：删除时弹出确认对话框，显示作品名、字数、对象数，需要输入作品名确认。

7. **空状态增强**：当前空状态已经存在但信息量低。v1.2 空状态增加快速链接：使用模板创建、导入、查看教程。删除「点击右上角创建」→改为更好的 CTA 按钮群组。

8. **作品统计条**：书架顶部显示统计 `共 N 部作品 | 总字数 XX 万 | 最高正典对象 M 个`

### 1.4 视觉示意

每张卡片 280px min，渐变背景保留并增强为**动态渐变**（悬停时渐变角度缓慢旋转动画，5s 周期）。底部信息行统一样式。右上角 hover 显示操作按钮。

卡片不再只有一个纯色渐变——改为包含对象头像星位图：在渐变背景的右上位置，根据作品中核心正典/项目正典对象的比例，显示 1-3 个小圆点（按正典等级着色：金/蓝/紫）。

## 2. 新建流程

### 2.1 当前问题

- 点击「新建作品」立即创建 `mapProjectToCreate('新作品')`，无任何输入
- 无体裁选择，一律「未分类」
- 默认渐变硬编码 `['#6366f1','#8b5cf6']`
- 用户只能稍后在假想的编辑功能中改名

### 2.2 设计方案

**新建向导（两步式，非强制——高级用户可跳过）：**

#### 方案 A：快捷弹窗

```
┌──────────────────────────────┐
│  新建作品                    │
│                              │
│  作品名: [______________]    │
│                              │
│  体裁: [科幻 ▼]             │
│                              │
│  初始设定模板:               │
│  ┌──────┐ ┌──────┐ ┌──────┐ │
│  │ 空白  │ │科幻   │ │奇幻   │ │
│  │ 建立  │ │ 模板  │ │ 模板  │ │
│  └──────┘ └──────┘ └──────┘ │
│  更多模板→                   │
│                              │
│         [取消]  [开始创作]   │
└──────────────────────────────┘
```

- 按 Enter 快速确认（作品名不为空时）
- 体裁 dropdown 预置常见体裁：科幻、奇幻、武侠、悬疑、言情、历史、都市、末世、恐怖、其他
- 初始设定模板可选：选择后自动创建一组预置对象（如科幻模板创建「世界观」「主要科技」「关键组织」三个草案对象）
- 点「开始创作」后：创建项目 + 导航到编辑器+自动聚焦第一个对象

#### 方案 B：快速创建（Power User）

按住 `Ctrl+点击「新建作品」` 直接创建，使用上次使用的体裁。跳过弹窗。

#### 交互细节

1. 弹窗聚焦时，输入框自动获得焦点
2. 体裁改变时，封面渐变色模板预览自动更新（科幻→蓝紫渐变，奇幻→金绿渐变）
3. 选择初始设定模板时，弹窗右侧显示模板预览卡片（列出会创建的预设对象）
4. 创建成功后显示短暂 toast「作品「觉醒纪元」已创建，已导入 3 个预设设定」
5. 如果用户取消创建，不留下任何空项目

## 3. 编辑器布局

### 3.1 当前问题

- 三种编辑模式（WYSIWYG/Source/Preview）用按钮切换，但区分度不够
- 无自动保存指示器——用户不确定内容是否已保存
- 属性条（类型/状态/正典）在工具栏下方，视觉权重过高且混乱
- 字数信息在底部，容易被忽略
- TipTap 编辑器的工具栏与源码模式工具栏内容不同，但 UI 风格一致，缺乏模式识别度
- Markdown 支持不足——WYSIWYG 模式下输入的 Markdown 符号不被解析
- 右侧 Inspector 与文档编辑区是同级关系，但 Inspector 内容可能与当前编辑对象不对应（Inspector 显示 selectedObject，编辑区显示 currentObject）

### 3.2 设计方案——「Markdown-First 编辑器」

核心转变：**WYSIWYG 模式降级，Source 模式升级为默认。**

v1.2 的默认编辑模式是 **美化源码模式**——所见即所写，但 Markdown 语法实时渲染为视觉样式（类似 iA Writer / Typora）：

```
┌──────────────────────────────────────────────┐
│ 大纲 |  # 张三                  模式: [源码|预览] │
│ 废弃  |                                        │
│ 草稿  |  ⬜ 类型: 人物    ⬜ 正典: 核心正典    │
│ 正文  |  ═══════════════════════════════════  │
│ 人物  |  ## 基本信息                           │
│  ·张三 |                                        │
│  ·李四 |  姓名：张三   年龄：28   身份：人造人  │
│ 设定  |                                        │
│       |  ## 外貌                               │
│       |  高个子，黑色短发，瞳孔带有微弱荧光...   │
│       |                                        │
│       |  [[乐园机制]] 是维持这个世界的...        │
│       |                             字数: 1,234 │
└──────────────────────────────────────────────┘
```

#### 三种模式的重新定义

| 模式 | 名称 | 用途 | 默认？ |
|------|------|------|--------|
| **源码** | 编辑模式 | 带语法高亮的 Markdown 编辑，实时渲染标题/粗斜体/引用 | **v1.2 默认** |
| **预览** | 阅读模式 | 纯净阅读视图，无编辑光标，不可编辑 | 否 |
| **WYSIWYG** | 富文本模式 | TipTap 所见即所得，适合排版操作 | 降级为可选 |

#### 设计亮点

**1. 源码高亮模式（默认）：**

- Textarea 改为 CodeMirror / Monaco-like 编辑器（或自定义语法高亮 div+hidden textarea）
- Markdown 语法实时着色：`# 标题` 显示为大字号粗体，`**粗体**` 显示为视觉粗体但仍保留 `**` 标记
- `[[WikiLink]]` 渲染为蓝色可点击标签，区别于纯文本
- 支持 Markdown 快捷输入：输入 `## ` 自动进入标题，`- ` 进入列表，`> ` 进入引用

**2. 自动保存指示器：**

```
[状态栏右侧]
保存中... ⟳    → 自动保存触发时显示
已保存 ✓       → 保存完成后显示 1.5s 后淡出
未保存 ●       → 有未保存修改时持续显示
```

- 每次内容变更触发 500ms debounce 后自动保存
- 保存完成后 toast 不阻断编辑，仅状态栏指示
- 离线/后端不可用时状态栏显示「离线 ●」

**3. 属性条压缩：**

- 属性从独占一行压缩到工具栏右侧
- 类型显示为图标 badge（人物👤/地点📍/组织🏛/规则⚙️/事件📅/物品📦/术语📖/章节📄）
- 正典等级显示为彩色条（金/蓝/紫/灰）
- 状态显示为圆点指示灯
- 点击 badge 展开 inline 编辑器修改属性（不再打开 Inspector）

**4. 字数/链接统计栏：**

- 固定在编辑器底部，灰色小字
- 显示：字数 | 链接数 | 正典等级 | 更新时间
- 双击链接数展开链接面板（列出所有 [[WikiLink]] 及其目标状态）

**5. 大纲左侧：**

- 当前大纲保留分组折叠结构
- 新增：大纲中正典等级用左侧色条表示（金/蓝/紫/灰）
- 新增：大纲搜索（大纲顶部搜索框，过滤对象名）
- 新增：拖动排序（支持在大纲中拖动对象到不同分组？v1.2 暂不做，留 v1.3）

**6. 创建/跟随 WikiLink 增强：**

- WikiLink 悬停显示 tooltip（对象类型+状态+摘要前 50 字）
- 悬停 tooltip 中包含「跳转」和「快速创建」按钮
- 不存在的 [[链接]] 用虚线橙色下划线 + 不同光标
- 双击已存在的链接：在当前页面中滑动到引用处（类似 VS Code 的 peek），而非跳转

### 3.3 自动保存设计

```
用户输入
  → 500ms idle debounce
  → 状态栏: "保存中..."
  → tauri-api.updateWorldObject()
  → 成功 → 状态栏: "已保存 ✓" (1.5s fade)
  → 失败 → 状态栏: "保存失败 ●" (持续，点击重试)
  → 切换对象时强制立即保存当前内容
```

### 3.4 Inspector 联动修复

- Inspector 始终显示 selectedObject（已实现）
- 编辑器显示 currentObject（已实现）
- 当用户在 Inspector 中点击不同对象时：同步更新编辑器的 currentObject
- 当编辑器切换到不同对象时：同步更新 Inspector 的显示
- **修复当前 Bug：** selectedObject 和 currentObject 去重，两者始终同步

## 4. 画板交互

### 4.1 当前问题

- 无缩放控件（state 中有 `scale/panX/panY` 但 UI 中无缩放按钮）
- 无框选（多选通过 Ctrl+点击，无矩形框选视觉反馈）
- 时间线无法拖拽重排
- 画板缩放到适应按钮已存在但功能不稳定
- 自动布局只有 force-directed，无网格/分层/径向选项
- 连线创建需要两步（点击源→拖拽到目标→选择类型），且连线拖拽时无实时预览蓝色指示线
- 画板间对象不能拖拽移动

### 4.2 设计方案

#### 缩放控件

在画板右下角添加浮动缩放控件：

```
┌──────────────────────────┐
│  [-]  100%  [+]  [⊞适应] │
└──────────────────────────┘
```

- `-` / `+` 以 25% 步进缩放（25% / 50% / 75% / 100% / 125% / 150% / 200%）
- 百分比显示当前缩放值
- `⊞适应` 按钮将画板缩放到适合视口
- 快捷键：`Ctrl+滚轮` 缩放，`Ctrl+0` 适应
- 缩放锚点为鼠标位置（类似 Figma）

#### 框选 (Marquee Select)

- 在 `select` 模式下，在空白区域拖拽画出一个矩形选区
- 选区使用半透明蓝色矩形（`rgba(144, 202, 249, 0.15)` + `#90CAF9 边框 dashed`）
- 完全被选区覆盖的节点变为选中状态
- 选中节点组可拖拽整体移动
- 选区在 mouseup 后消失，节点保持选中
- 选中组在外部点击空白区域时取消全选

#### 时间线拖拽重排

- 时间线上的事件节点可以从左侧「待排事件区」直接拖拽到时间线主轴上
- 拖拽到主轴上时，插入点显示绿色插入指示线
- 时间线节点之间可以拖拽重新排序（拖拽时显示占位线）
- 时间线节点右侧显示「关键转折点」星标 toggle
- 时间线缩放：水平方向可缩放时间跨度（周/月/季/年视图切换）

#### 画板工具侧栏改进

当前 8 个工具按钮保留，布局优化：

- 工具按钮分组：选择相关（选择/多选/框选）/ 创建相关（对象池/模板/文本/便签）/ 连线相关（连线/分区）/ 视图相关（适应/布局）
- 激活态工具使用主色 `#B7FF00` 填充背景
- 新增「撤销」和「重做」按钮（画板操作历史）

#### 连线交互改进

- 连接源节点拖出连线时，目标节点显示半透明高亮（drop target glow）
- 连线类型选择从弹出面板改为：在连线上方显示**内联 quick-pick bar**
  - 显示 8 种连线类型为紧凑标签行，点击即选
  - 预设根据源/目标类型智能排序（如人物→人物默认显示「相关/影响/冲突」）
- 连线创建后自动添加类型标签到连线中央

#### 对象池改进

- 对象池现在可以从画板侧栏直接展开为**浮动抽屉**（不遮挡整个画布）
- 对象池搜索即时过滤
- 从对象池拖拽对象到画板：从抽屉拖出到目标位置，对象自动定位在释放点

#### 画布无限扩展

- 当前画布有边界限制。v1.2 移除边界，改为无限滚动
- 背景网格随缩放调整密度（缩放级别低时网格线间距自动增大）
- 超出初始视口时画布自动扩展，不再需要「适应画布」才能看到所有内容

## 5. 正典引导

### 5.1 当前问题

- 正典等级存在于 type 系统（`CanonLevel = '未收录' | '草案正典' | '项目正典' | '核心正典'`），但用户界面上只有 dropdown 选择
- 无首次使用提示 —— 用户不知道「正典」是什么
- 无正典体系的可视化展示，用户不理解四个等级的含义
- 操作「收录为设定」的触发路径单一（仅 Inspector 面板中的按钮）

### 5.2 设计方案——四层正典可视体系

#### 正典等级视觉化

每个对象的正典等级在 UI 中**处处可见**，不再只是 dropdown 中的文字：

| 等级 | 视觉标识 | 出现位置 |
|------|---------|---------|
| 核心正典 | 金色实心圆 `●` + 金色边框 | 大纲、卡片、画板节点、Inspector |
| 项目正典 | 蓝色实心圆 `●` + 蓝色边框 | 同上 |
| 草案正典 | 紫色实心圆 `●` + 紫色边框（虚线） | 同上 |
| 未收录 | 灰色空心圆 `○` + 灰色边框（点线） | 同上 |

**圆形徽章**统一显示在对象名左侧，作为快速视觉参考。彩色边框作为节点和卡片的外围辅助识别。

#### 首次引导——「正典入门」卡片

当用户第一次创建对象时（或第一个对象状态变为非「草稿」时），自动弹出 **世界构建引导卡片**：

```
┌──────────────────────────────────┐
│  🌐 世界构建入门                  │
│                                   │
│  织梦机用「正典」来管理你的设定    │
│  稳定性。每个设定有四个等级：      │
│                                   │
│  ● 未收录 — 灵感笔记，尚未确认    │
│  ● 草案正典 — 初步认定，还在打磨  │
│  ● 项目正典 — 团队认可，纳入正典  │
│  ● 核心正典 — 不可更改的基石设定  │
│                                   │
│  [✓ 不再显示]     [进入手册 →]     │
│                        [开始使用]  │
└──────────────────────────────────┘
```

- 仅首次访问项目时显示（localStorage 标记）
- 「进入手册 →」打开正典完整手册（独立页面）
- 「开始使用」关闭卡片，进入正常编辑

#### 正典手册（独立页面）

在导航栏「设定集」旁新增「正典」tab（或作为设定集的子视图）:

```
┌──────────────────────────────────────┐
│  正典手册                            │
│                                      │
│  ┌──                                │
│  │  🌟 核心正典                      │
│  │  定义：不可更改的基石设定。       │
│  │  一旦锁定为核心正典，修改需要     │
│  │  经过完整的正典裁决流程。         │
│  │  当前: 2 个对象                   │
│  │  [张三] [乐园机制]               │
│  └──                                │
│                                      │
│  ┌──                                │
│  │  📘 项目正典                      │
│  │  定义：团队/个人认可的正典设定。  │
│  │  修改需要记录判断原因。           │
│  │  当前: 0 个对象                   │
│  └──                                │
│                                      │
│  ┌──                                │
│  │  🟣 草案正典                      │
│  │  定义：初步认定，还在打磨。       │
│  │  可以自由修改。                   │
│  │  当前: 3 个对象                   │
│  │  [替换计划] [人造人组织] [...]    │
│  └──                                │
└──────────────────────────────────────┘
```

点击对象名直接导航到该对象的文档视图。

#### 正典操作强化

v1.2 新增正典操作入口：

1. **收录为设定**（已有，保留）— 将未收录→草案正典
2. **提升正典**（已有，保留）— 向上升一级
3. **降级正典**（已有，保留）— 向下降一级
4. **正典裁决**（新增）— 核心正典变更必须经过裁决流程（输入原因+确认）
5. **批量收录**（新增）— 在大纲中多选对象，批量收录/提升/降级

#### 判断记录的可视化

- 判断记录从文本日志改为**时间线视图**（类似 Git 提交历史）
- 每个判断显示为时间线上的一个节点，颜色区分操作类型
- 时间线中显示操作前后的比较（前→后）
- 在 Inspector 中，判断记录最小化显示为统计数字（+ 展开按钮）

### 全局快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+N` | 新建作品（书架），新建对象（编辑器） |

| `Ctrl+,` | 进入设置/主题切换 |
| `Esc` | 关闭弹窗/取消选择/退出编辑 |

### 导航流程

```
书架
  ├─ 点击作品 → 编辑器（文档 tab，默认选中第一个对象）
  │   ├─ Tab 切换: 文档 ↔ 画板 ↔ 设定集 ↔ 判断记录/正典
  │   ├─ 点击「← 书架」→ 返回书架
  │   └─ 双击 WikiLink → 切换到目标对象（当前 tab 保持）
  │
  ├─ 点击新建 → 新建向导弹窗 → 确认后进入编辑器
  │
  └─ 搜索/筛选 → 即时过滤卡片网格
```

### 响应式行为

- 窗口 < 768px：隐藏 Inspector，大纲自动收起，画板侧栏变为底部栏
- 窗口 < 480px：书架卡片改为单列布局，新建按钮改为浮动 FAB
- 窗口 > 1400px：Inspector 和 大纲 同时显示不变

### 色彩对比度规范

所有主题必须满足 WCAG AA 标准：
- 正文对比度 ≥ 4.5:1
- 大字号标题 ≥ 3:1
- 边框和装饰元素 ≥ 3:1（非必需内容）

### 主题切换入口

- 设置面板（`Ctrl+,`）中的主题选择器
- 书架底部或「更多」菜单中的主题切换
- 主题选择立即生效，不刷新页面
- 主题选择持久化到 localStorage

## 附录：设计差异对照表

| 功能 | v0.2 现状 | v1.2 目标 | 差异化理由 |
|------|-----------|-----------|-----------|
| 编辑模式 | WYSIWYG 默认 | 源码高亮默认 | 世界构建者需要精确控制 Markdown，对应工程师更喜欢源码 |
| 对象关系 | 画板连线+WikiLink | 画板连线+WikiLink+正典等级视觉化 | 正典是织梦机核心概念，必须前台化 |
| 新建流程 | 一键创建「新作品」 | 两步向导+模板 | 降低"空白页恐惧"，同时提供快速跳过 |
| 画板缩放 | 无 UI | 右下角浮动控件 | 基础操作，缺失不可接受 |
| 正典引导 | 无 | 首次引导卡片+手册+视觉徽章 | 无引导的概念系统等于不存在 |
| 保存反馈 | 无 | 状态栏指示器 | 创作者最关心数据安全 |
