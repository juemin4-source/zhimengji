# 竞品调研：Obsidian Canvas 与 Figjam 画板交互

## 概述

本文调研两个代表性画板工具：**Obsidian Canvas**（知识管理场景的白板）和 **Figjam**（协作设计场景的白板）。两者都提供无限画布 + 卡片 + 连线的核心交互，但面向不同场景，其设计取舍对织梦机的画板交互有直接参考价值。

织梦机当前的画板实现了基础节点（StoryNode）、连线（BoardEdge）、位置（x/y）、缩放与布局，但交互深度、AI 集成和协作能力与竞品存在明显差距。

---

## 第一部分：Obsidian Canvas 核心交互

### 1. 卡片（节点）系统

Obsidian Canvas 是 Obsidian 的官方核心插件，2023 年发布后持续迭代。

**支持的卡片类型：**

| 卡片类型 | 创建方式 | 说明 |
|----------|----------|------|
| 文本卡片 | 双击空白处 | 内联编辑，支持 Markdown、代码块、链接 |
| 笔记卡片 | 从文件管理器拖入 | 显示 vault 中任意笔记的内容预览 |
| 媒体卡片 | 拖入图片/PDF/音频/视频 | 支持直接在画板中预览 |
| 网页卡片 | 右键 → 添加网页 | 嵌入 URL 预览 |
| 文件夹卡片 | 拖入文件夹 | 一次添加整个文件夹的所有文件 |

**关键交互：**
- 双击卡片内联编辑（不跳出画板）
- 右键 → 替换文件（Swap file）
- 卡片大小可调整
- 支持三种尺寸模式

### 2. 连线（边）系统

**连接方式：**
- 鼠标悬停在卡片边缘 → 从圆点拖出连线到另一卡片
- 连线可添加标签（双击连线）
- 连线可着色（选中后使用调色板）
- 右键连线 → 跳转源/目标节点（在大画板上非常有用）
- 连线端点可重新拖拽连接

**连线交互深度：**
Obsidian Canvas 的连线相对简单（无箭头方向、无样式多样性）。社区插件 **Advanced Canvas** 弥补了这些短板：
- 多种箭头样式
- 路径样式（实线/虚线/点线）
- 智能路由（自动绕开节点）
- 浮动锚点（自动选择连线端点）
- 快速反向连线方向

### 3. 分组系统

- 选中多个卡片 → 创建分组
- 分组支持命名（双击组名称）
- 分组可嵌套（组内含组）
- 分组可着色
- **Advanced Canvas 增强**：可折叠/展开分组（减少视觉杂乱）

### 4. 导航与交互

| 交互 | 操作 |
|------|------|
| 平移 | 按住 Space + 拖拽（或滚动） |
| 缩放 | Ctrl/Cmd + 滚轮 |
| 适配视图 | 缩放至全部卡片 / 缩放至选中卡片 |
| 网格吸附 | 拖动卡片自动对齐（按住 Space 可临时禁用） |
| 聚焦模式（Advanced Canvas） | 选中节点，模糊其他全部内容 |
| 演示模式（Advanced Canvas） | 将画板转为幻灯片演示 |

### 5. 文件格式与互操作性

Obsidian Canvas 使用 **JSON Canvas 格式**（`.canvas` 文件），这是一个开放格式：
- 纯 JSON，易于脚本处理
- 支持 Git 版本控制
- 可在 Markdown 笔记中嵌入：`![[filename.canvas]]`
- 支持画板嵌套
- 程序化创建/修改卡片和连线

### 6. Advanced Canvas 插件（2024 亮点）

这个社区插件（Developer-Mike 开发，683+ stars）大幅扩展了 Canvas 能力：

| 功能类别 | 增强内容 |
|----------|----------|
| **元数据** | 完整集成 Graph View、Backlinks、Outgoing Links |
| **节点样式** | 流程图形状（圆角、菱形、平行四边形）、边框样式、自动调整大小 |
| **连线增强** | 路径样式、箭头样式、智能路由、浮动边、反向 |
| **交互增强** | 可折叠分组、演示模式、嵌入另一画板（Portal）、焦点模式、边的悬停高亮 |
| **嵌入能力** | 支持在 Markdown 中链接/嵌入画板中的单个节点：`[[canvas-file#node-id]]` |
| **导出** | PNG/SVG 导出支持透明背景 |

## 第二部分：Figjam 核心交互

### 1. 便签与元素系统

Figjam 是 Figma 旗下的协作白板工具，2020 年发布，2023-2024 年大规模加入 AI 能力。

**核心元素：**
- **便签（Sticky Note）**：画板基础元素，多色可选，内联编辑
- **形状（Shapes）**：矩形、圆形、菱形、箭头等
- **连线（Connectors）**：带箭头/不带箭头、直线/曲线
- **文字（Text）**：独立文本框
- **图片（Images）**：拖入即可
- **便签板（Sticky Board）**：预定义的模板区域
- **Widget 系统**：可嵌入第三方交互组件

### 2. 画板交互深度

**协作核心：**
- **多人实时编辑**：每个协作者有颜色光标
- **评论系统**：在任意元素上添加评论
- **@提及**：@团队成员通知
- **同步状态指示**：谁在看哪里、谁在编辑什么

**组织能力：**
- **分组（Group）**：选中多个元素一键分组
- **区域（Section）**：命名背景区域，划分画板功能区
- **锁定（Lock）**：锁定元素位置/内容防止误改
- **网格对齐**：拖动吸附 + 对齐线
- **任何位置添加便签**：双击即创建

**模板系统：**
- 丰富的预置模板库（Sprint 规划、用户故事映射、流程图、组织结构图、头脑风暴等）
- 社区模板市场
- AI 模板生成（2024 新增）

### 3. AI 能力概览（详见 figjam-ai-research.md）

- AI 模板生成（自然语言 → 结构化画板）
- 便签自动分组/排序
- 头脑风暴摘要
- 流程图/时间线生成
- Jambot Widget（类 ChatGPT 画板助手）

### 4. Figjam vs Obsidian Canvas：核心差异

| 维度 | Obsidian Canvas | Figjam |
|------|----------------|--------|
| **设计目的** | 个人知识可视化 | 团队协作白板 |
| **协作** | 无实时协作（只读分享） | 多人实时协作核心 |
| **AI 深度** | 无原生 AI | 模板生成 + 便签分组 + 摘要 |
| **文件格式** | JSON Canvas（开放） | 私有格式 |
| **与编辑器的集成** | 深度集成（拖拽笔记、嵌入内容） | 无编辑器集成 |
| **节点内容** | 可展示完整笔记内容 | 便签文本为主 |
| **自定样式** | CSS snippets + Advanced Canvas | 有限内置样式（颜色/字体） |
| **性能** | 大画板时可能卡顿 | 企业级性能 |
| **离线** | 完全离线 | 需要网络（有桌面端） |
| **插件生态** | 丰富的社区插件 | Widget 市场 |

## 第三部分：织梦机画板与竞品的差距分析

### 织梦机当前画板能力（基于代码分析）

从织梦机源码中可以看到当前画板的基础能力：

- 节点（StoryNode）：位置 x/y、类型（plot/character_beat/twist/foreshadow/note）、尺寸模式（compact/standard/wide）
- 连线（BoardEdge）：源/目标 ID、关系类型（related/influences/conflicts/supports/replaces/hostile 等 12 种）、标签、样式
- 画板（Board）：类型（free/character_relation/timeline/deduction/chapter_layout 等 9 种）
- 缩放控制：0.2x - 3.0x，步进 1.3x
- 自动布局：cose 算法（来自 Cytoscape）
- 小地图：右下角
- 搜索/筛选：按查询和类型筛选
- 连线弯曲点（EdgeBendPoint）
- 连线标签编辑

### 与 Obsidian Canvas 的差距

| 缺失能力 | 优先级 | 说明 |
|----------|--------|------|
| **卡片内联编辑** | 高 | 目前画板节点和编辑器分离，用户不能在画板内直接编辑内容 |
| **卡片内容预览** | 中 | 无法在画板卡片中看到叙事单元的全文或摘要 |
| **分组系统** | 高 | 没有视觉分组能力，大量节点时难以组织 |
| **可折叠嵌套** | 中 | 没有子画板/节点展开概念 |
| **富媒体支持** | 中 | 不支持图片预览、音频等 |
| **演示模式** | 低 | 画板内容丰富后可转为故事展示 |

### 与 Figjam 的差距

| 缺失能力 | 优先级 | 说明 |
|----------|--------|------|
| **实时协作** | 低（v2.0+）| 织梦机 v1.X 是单人工具 |
| **AI 辅助** | 高（v1.6）| 自动布局、聚类、一致性扫描 |
| **模板系统** | 中 | 缺少画板类型对应的布局模板 |
| **评论/标注** | 低 | 当前版本为单人模式 |
| **便签快速创建** | 高 | 双击空白处创建新节点的交互 |

## 第四部分：可借鉴的设计模式

### 1. 双重交互模式：编辑模式 + 浏览模式

借鉴 Obsidian Canvas 的"双击编辑"和 Figjam 的"点击选择、双击编辑"，织梦机画板应支持：
- **浏览模式（默认）**：点击选中、拖拽移动、连线可见
- **编辑模式（双击）**：内联编辑节点内容，支持 Markdown 快捷键
- 模式切换应该流畅无缝，不打断思维流

### 2. 可视化的分组 + 折叠机制

织梦机的 9 种画板类型（角色关系、时间线、伏笔、逻辑循环等）天然适合可视化分组：
- **按类型自动着色**：不同类型的节点默认颜色不同
- **区域（Section）**：像 Figjam 的背景区域，可以用矩形背景标记"第一幕""第二幕"等
- **折叠分组**：Advanced Canvas 的可折叠行为对大规模画板极其有用

### 3. 连线语义化

织梦机已有 12 种关系类型，但交互没有充分利用这一语义：
- **连线着色 + 样式**：已实现（LINK_TYPE_STYLES），但需要更直观的选择 UI
- **连线过滤器**：按关系类型筛选显示/隐藏连线
- **连线图例**：画板一角显示类型图标/颜色对照表
- **连线路径智能路由**：当节点密集时，连线应自动绕开

### 4. 导航增强

- **画板级别跳转**：从大纲树可直接跳转到特定画板的特定节点
- **节点搜索 + 聚焦**：在当前画板中搜索节点名称，自动聚焦并高亮
- **历史后退/前进**：像 Obsidian Canvas 一样支持导航历史

### 5. 信息密度自适应

Obsidian Canvas 有节点尺寸切换，但织梦机可以更进一步：
- **缩放级别驱动信息密度**：缩小到 0.5x 以下时，节点只显示标题 + 类型图标；放大到 1.5x 以上时显示内容摘要
- **展示模式（Presentation）**：用于向他人展示故事结构

## 第五部分：差异化策略

### 织梦机不适合直接复制的功能

1. **Obsidian Canvas 的文件系统映射**：织梦机的叙事单元是结构化对象，不是文件。不需要将画板节点映射到文件系统。

2. **Figjam 的实时多人协作**：v1.X 阶段织梦机是单人桌面工具。协作是 v2.X 的方向，不应该在早期版本引入复杂度。

3. **完全通用的白板**：织梦机画板是"叙事结构画板"，不是通用白板。它应该有叙事意图——每种画板类型预置了特定的交互和语义。

### 织梦机的画板差异化方向

1. **叙事类型画板**：织梦机的 9 种画板类型本身就是差异化——它们不是通用白板的子集，而是针对叙事结构设计的专业视图：
   - 角色关系画板预置角色节点样式和关系类型
   - 时间线画板预置时间轴交互和事件排序
   - 伏笔画板预置伏笔-揭示连线类型

2. **语义连线系统**：竞品的连线只是视觉连线。织梦机的 12 种关系类型是**结构化数据**——它们会影响画板布局算法、一致性检查、叙事分析。这不是视觉装饰，是叙事模型的组成部分。

3. **画板 ↔ 编辑器双向同步**：在画板上修改节点位置 → 编辑器中的章节出现；在编辑器中创建新章节 → 画板自动出现对应节点。这种数据层面的同步是竞品没有的深度集成。

4. **AI 驱动的画板操作**：
   - AI 建议节点分组方式（"这些角色有共同线索，建议分组"）
   - AI 检测缺失连线（"节点 A 和节点 B 有引用关系但未连线"）
   - AI 自动布局不只是一次性算法，而是持续的面包屑导航

5. **节奏可视化**：在画板中叠加叙事节奏曲线（张弛幅度），让用户在空间布局上看到故事的情感节奏，这是任何通用白板都没有的能力。

6. **组合视图**：允许用户在一个画板中嵌入另一个画板（类似 Advanced Canvas 的 Portal），但带语义关联——子画板的内容变化会反映到父画板的摘要指标上。

---

## 总结

Obsidian Canvas 和 Figjam 代表了"个人知识白板"和"团队协作白板"两种范式的顶端。织梦机画板不需要在通用性上追赶它们，而应该在**叙事专业度**上建立壁垒：

- Obsidian Canvas 好，但对于叙事者不够专业
- Figjam 好，但对于单人写作太重了
- 织梦机画板应该成为：**为叙事者设计的、带 AI 辅助的、语义驱动的结构可视化工具**
