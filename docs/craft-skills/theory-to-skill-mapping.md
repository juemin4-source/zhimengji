## 一、审美篇 → Craft Skills

> 核心命题：审美不是"好不好看"，而是"故事作为整体是否成立"
> 口诀：不是"好不好"，而是"够不够"

### CS-01: craft-aesthetics-concrete（具象诊断）

| 字段 | 内容 |
|------|------|
| **理论出处** | 审美篇 — 关键词「具体」 |
| **原文** | "不写笼统的'他很难过'，写'他把杯子转了四圈'" |
| **功能** | 检查文本是否将抽象情感/状态转化为具体动作、物色、场景 |
| **输入** | 文本段落 / 场景描述 |
| **输出** | 具象化评分 + 建议替换的抽象词列表 |
| **调用链** | CS-01 → CS-04(得度) → CS-32(熔裁) |
| **用户深度** | 小新:自动标记抽象句; 老张:查看具体建议; 老陈:逐句对照调整 |
| **不适用于** | 诗歌、非叙事性文本、纯对话场景 |
| **验证方法** | 随机抽样标记的抽象句确认准确性；抽查未被标记的描述性段落验证无漏报 |

### CS-02: craft-aesthetics-complexity（复调诊断）

| 字段 | 内容 |
|------|------|
| **理论出处** | 审美篇 — 关键词「复杂」 |
| **原文** | "不是多几条支线，而是同一件事在不同人眼里有不同的重量" |
| **功能** | 检测叙事是否在多视角/多层面上有重量差异，而非平行堆砌 |
| **输入** | 章节文本 + 人物表 |
| **输出** | 视角重量分布图 + 单薄视角警告 |
| **调用链** | CS-02 → CS-21(narrative-entry) |
| **用户深度** | 小新:自动检测; 老张:查看视角分布; 老陈:手动标注关键事件的多视角权重 |
| **不适用于** | 单视角叙事、非叙事文本 |
| **验证方法** | 同一事件在多视角下交叉验证权重差异 |

### CS-03: craft-aesthetics-reserve（余地诊断）

| 字段 | 内容 |
|------|------|
| **理论出处** | 审美篇 — 关键词「余地」 |
| **原文** | "不要把所有东西都解释清楚。留出让读者参与的空间" |
| **功能** | 检测解释过度区域，标记"说尽"的段落 |
| **输入** | 文本段落 |
| **输出** | 过度解释标记 + 余地评分 |
| **调用链** | CS-03 → CS-35(depth-reserve) |
| **用户深度** | 小新:自动标记; 老张:查看过度解释段落; 老陈:逐句裁减 |
| **不适用于** | 诗歌、意识流文本、极短篇 |
| **验证方法** | 对比删除标记段后的可读性变化；确认读者能自行推断 |

### CS-04: craft-aesthetics-measure（得度检查）

| 字段 | 内容 |
|------|------|
| **理论出处** | 审美篇 — 关键词「得度」 |
| **原文** | "不多不少。每个场景、每句话、每个物色，都恰好在它应该在的位置" |
| **功能** | 综合检查场景/对话/描写的长度是否与叙事功能匹配 |
| **输入** | 整章文本 + 场景划分 |
| **输出** | 场景长度/功能匹配报告 + 冗余场景标记 |
| **调用链** | 上游:CS-01~03 → CS-04 → 下游:CS-35(熔裁) |
| **用户深度** | 小新:自动报告; 老张:查看建议; 老陈:确认裁切 |

> **三重检验**已合并到 CS-38 (craft-gate-triple-test)。本处不再单独定义。

## 二、读者篇 → Craft Skills

> 核心命题：读者不是被动接收者
> 一句话：不是读者看不懂，而是你没给读者看下去的理由

### CS-05: craft-reader-desire-entry（欲望入口诊断）

| 字段 | 内容 |
|------|------|
| **理论出处** | 读者篇 — 「欲望入口」 |
| **原文** | "故事必须在早期给读者一个'想知道接下来怎样'的欲望" |
| **功能** | 检测开篇是否建立了有效的欲望入口（追问/悬念/情感/命运） |
| **输入** | 开篇1-3章文本 |
| **输出** | 欲望入口类型识别 + 入口强度评分 + 缺失警告 |
| **调用链** | CS-05 → CS-06(low-entry) → CS-07(genre-fidelity) |
| **用户深度** | 小新:自动检测; 老张:查看类型分析; 老陈:手动调整入口策略 |

### CS-06: craft-reader-low-entry（低入口高经验诊断）

| 字段 | 内容 |
|------|------|
| **理论出处** | 读者篇 — 「低入口，高经验」 |
| **原文** | "读者不需要大量知识就能进入，但离开时会带着丰富的经验" |
| **功能** | 检测设定/背景信息的释放节奏，是否在早期存在过高的认知门槛 |
| **输入** | 开篇 + 世界观设定文档 |
| **输出** | 认知负荷曲线 + 信息密度警告 + 潜在入口障碍标记 |
| **调用链** | CS-06 → CS-20(info-pacing) |
| **用户深度** | 小新:自动生成认知曲线; 老张:查看障碍点; 老陈:调整信息分布 |

### CS-07: craft-reader-genre-fidelity（类型承诺检查）

| 字段 | 内容 |
|------|------|
| **理论出处** | 读者篇 — 「类型承诺」 |
| **原文** | "你在开篇承诺了什么类型，就要交付什么类型" |
| **功能** | 检查全文的类型一致性，标记偏离类型承诺的段落 |
| **输入** | 全文文本 + 用户声明的类型标签 |
| **输出** | 类型偏离标记 + 类型一致性评分 |
| **调用链** | CS-07 可调用其他所有 skill 检查类型一致性 |
| **用户深度** | 小新:自动检查; 老张:查看偏差报告; 老陈:确认类型定位 |

### CS-08: craft-reader-info-pacing（信息释放节奏）

| 字段 | 内容 |
|------|------|
| **理论出处** | 读者篇 — 读者层诊断要点 |
| **原文** | "有没有让读者等太久？有没有替读者做完所有判断？有没有在读者已经知道的时候还在解释？" |
| **功能** | 分析信息释放的频率和时机，检查"过迟"和"过早"的问题 |
| **输入** | 章节文本 + 关键信息列表 |
| **输出** | 信息释放时间线 + 延迟警告 + 冗余解释标记 |
| **调用链** | CS-08 → CS-03(reserve) → CS-04(measure) |
| **用户深度** | 小新:自动时间线; 老张:查看节奏; 老陈:调整信息释放点 |

### CS-09: craft-reader-audience-diagnosis（读者层综合诊断）

| 字段 | 内容 |
|------|------|
| **理论出处** | 读者篇 — 读者层诊断五点 |
| **原文** | "读者从哪里产生阅读欲望？这个欲望是否持续被喂养？信息释放节奏是否符合类型预期？" |
| **功能** | 综合五点诊断的读者层全面评估 |
| **输入** | 全文文本 |
| **输出** | 读者体验报告（欲望曲线/信息节奏/判断空间/冗余解释） |
| **调用链** | 上游:CS-05~08 → CS-09(综合) → CS-04(得度) |
| **用户深度** | 小新:一键报告; 老张:查看分布; 老陈:逐项确认 |

## 三、人物篇 → Craft Skills

> 核心命题：人物不是属性列表。人物是从机制中长出来的。
> 关键词：私/处/执/为/已 | 机制受力八位置

| 字段 | 内容 |
|------|------|

| **原文** | "人物首先从故事机制的受力点里来" |

## 三路用户介入深度

| 用户 | 介入模式 | 适用 skill |
|------|---------|-----------|
| **小新**（新手作者） | 自动运行，一键报告，不修改原文 | 所有 skill 均可自动运行，输出简化报告 |
| **老张**（积累型作者） | 查看建议，选择性修改 | 可查看各 skill 的详细分析，手动确认关键修改 |
| **老陈**（资深作者） | 手动控制每个维度，逐项确认 | 可深度介入每个 skill 的每个子维度，手动调整参数 |

同一套 skill 底层代码，不同 user 走不同 depth level：

```yaml
depth_levels:
  xiaoxin: 1  # 自动检测 + 一键报告
  laozhang: 2 # 查看详情 + 确认修改
  laochen: 3  # 手动控制 + 逐项调整
```
